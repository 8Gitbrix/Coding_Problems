Machine Problem 3: Sort Algorithm Timing

Objectives

  1. Use the timeit module to determine execution time
  2. Systematically record and tabulate the actual runtimes of different sorting algorithms
  3. Create a graph based on the aforementioned results

Reminder: Before starting ...

... you'll need to do (git fetch upstream ; git merge upstream/master) within your cloned repository to fetch my additions (consisting of template code and recent lecture examples).

Overview

For this machine problem you will use the timeit module to determine the runtime performance of at least 4 different sort algorithms. They will include:

  1. Insertion sort.
  2. "Divide" sort (presented in class as an intro to the divide-and-conquer idea).
  3. Either Bubble or Selection sort (links are to the relevant sections in the textbook, which contain code for their implementations).
  4. Merge sort.

Your finished program will run each of the sort functions on randomly shuffled lists of sizes ranging from 100 up to at least 1000 (you can go higher, if you've the time and patience) and produce a table relating input size to time taken.

Using the table, you will also create a line chart as shown below to visualize the data. Here's the Google Doc spreadsheet containing the (randomly generated) sample data and chart --- note that the raw table text is in the first column of the spreadsheet. Feel free to delete or tweak the trendlines.

Implementation Details

You will find template files for your implementation in the mps/03/ folder of your code repository. There are two files provided:

  1. sort_algs.py, which contains three of the four sort algorithms you will be timing. You should add your own sort function(s) to the end of this file.
  2. timing.py, which will contain the code that actually performs the timing. Note that it already contains some code that demonstrates how to use the timeit and random modules and string format function together to time insertion sort on a 100-element list and print out the elapsed time.

Using the provided code as a starting point, you should implement a loop in timing.py that runs all your sort algorithms on different input list sizes (using timeit), and produces output that looks like the following (note that the numbers below are made up --- yours will be quite different!):
