Machine Problem 7: Balanced Binary Search Trees

Objectives

  1. Use the recursive-helper-function pattern to implement binary search tree functions (insertion, deletion, search, etc.)
  2. Understand how to use rotations to maintain a balanced binary search tree (specifically, an AVL tree)
  3. Understand how methods annotated with @staticmethod differ from regular methods in Python
  4. Complete the balanced binary search tree implementation.

Reminder: Before starting ...

... you'll need to do (git fetch upstream ; git merge upstream/master) within your cloned repository to fetch my additions (consisting of template code and recent lecture examples).

Overview

For this machine problem you will complete the implementation of the balanced binary search tree implementation started in lecture. The base code can be found in the mps/07/bstree.py file.

You will need to add code to the following methods:

  • add
  • rebalance
  • contains
  • remove

Your add and remove implementations must, in the end, always leave the binary search tree strictly balanced --- such that all nodes in the tree have subtrees differing in height by a maximum of 1. We recommend, however, that you start with an unbalanced remove implementation first (to ensure correctness) before moving onto balancing --- you may add additional methods (static or not) to help distribute the workload.

The textbook's description of AVL Trees should prove helpful, though the book makes use of a "balance factor" instead of directly comparing heights of subtrees as we did in class. The Wikipedia section on deletion-rebalancing may also be helpful (especially for visualizations), though the implementation shown there is an iterative (as opposed to recursive) one.

For your convenience, the default __repr__ method of both the BSTree class itself and of its constituent Nodes make use of the _tree_repr (breadth-first) method. This should make it much easier to test and debug your work at the REPL; if you want to test your code on larger inputs you may wish to uncomment the following line in __repr__ to use the list-based representation (also comment out the original line, of course):

return repr(list(self))

Extra Credit

For extra credit (up to 20% additional points on this machine problem @ 10% each), you can implement the following additions:

  1. Instead of calling the height method everytime the height of a subtree is needed, add a height attribute to each node, which is populated on creation and only updated as necessary (i.e., on rotations, insertions, and deletions). This attribute will allow the height method to be entirely removed.

  2. Modify the _tree_repr method so as to improve the tree visualization; e.g., by incorporating ASCII art to indicate how nodes are related. The following might be the updated output for a small, balanced tree:

                 5
        /----+----\
       2           7
    /--+        /--+--\
   1           6       9
